Python Analysis on Loves Truck Stop Historical Spend Data

This project focuses on analyzing a large dataset of historical spend data from Loves Truck Stop to uncover patterns and insights that could inform pricing and operational strategies. The dataset was processed and analyzed using Pandas for data manipulation and transformation, and NumPy for efficient computation, especially when dealing with large volumes of data. The project specifically applied a Difference of Differences analysis, comparing a treatment group with a control group to evaluate the impact of certain operational changes. The computational demands of this analysis were handled efficiently using NumPy, which is well-suited for high-performance numerical calculations. To visualize the results and make them more accessible, Plotly was used for interactive data visualizations, allowing stakeholders to explore the data through dynamic charts and graphs. Additionally, mlxtend was utilized for Apriori analysis, which helped identify frequent itemsets and associations within the spend data, offering insights into customer behavior and purchasing patterns. This combination of technologies enabled the project to derive actionable business insights from complex data in a timely and efficient manner.

AWS Middleware Application for Data Synchronization

This AWS-hosted middleware application connected a Risk Management Platform API with a Vendor Management Platform to ensure synchronization of vendor profiles. The solution utilized AWS Step Functions, comprising AWS Lambda functions for specific transformations and validations. It integrated two platforms that communicated in different formats (JSON and XML), resolving the mismatches and ensuring seamless data synchronization through an API Gateway. The middleware was built to bridge the gap between these platforms, automating the synchronization process. The API Gateway exposed the middleware as a RESTful API, allowing communication with the external platforms. This project leveraged serverless architecture, utilizing AWS services to streamline development and reduce infrastructure overhead. The use of Lambda functions and Step Functions ensured that the solution was scalable, efficient, and easy to maintain, automating the synchronization of data between the two platforms.

Entity Resolution Solution with Python

This Python-based project focused on solving entity resolution challenges by clustering a large dataset using TF-IDF (Term Frequency-Inverse Document Frequency) to reduce time complexity. After clustering, a rule-based Bayesian Entity Matching technique was applied to perform entity resolution. This approach used probabilistic models to assess the likelihood that two records referred to the same entity based on their attributes. The final step in the process involved applying fuzzy matching to refine the entity resolution process, especially for cases where records contained slight variations in spelling or formatting. This combination of techniques ensured that the final dataset was accurate, with duplicate records successfully merged and inconsistencies resolved. This solution significantly improved data quality for downstream analytics and reporting. The project used Python’s powerful libraries and techniques to handle large datasets and deliver clean, accurate data for further analysis.
